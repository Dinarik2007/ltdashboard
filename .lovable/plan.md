## План: кнопка «Отчёт» с настраиваемым составом

Сейчас кнопка «Отчёт» в `src/components/topbar.tsx` ничего не делает. Сделаем её рабочей: пользователь сам собирает отчёт через диалог с галочками — выбирает период, формат и какие секции включить.

### UI: диалог «Сформировать отчёт»

Клик по «Отчёт» открывает модалку (shadcn `Dialog`) со следующими блоками:

1. **Период** — radio-группа: 7 дней / 30 дней / 90 дней / С начала года. Предзаполнен текущим значением из селектора периода в Topbar.
2. **Формат файла** — radio: PDF / CSV.
3. **Состав отчёта** — список чекбоксов (по умолчанию все включены):
   - KPI (Расходы, Охват, Подписчики, ROMI)
   - Расходы по каналам
   - Охват по неделям
   - Подписчики по платформам
   - ROMI по кампаниям
   - Лучшие каналы
   - Топ SKU
   - Соцпосты
   - Бюджеты (план vs факт)
4. **Доп. опции** — чекбоксы:
   - Добавить шапку с логотипом и датой генерации
   - Включить сводку (summary) в начале PDF
5. Внизу: кнопки «Отмена» и «Скачать» (disabled, если ни одна секция не выбрана).

После скачивания — toast `sonner` «Отчёт готов».

### Технически

- Новый файл `src/components/report-dialog.tsx` — диалог с формой (useState для period / format / sections / options).
- Новый файл `src/lib/report-export.ts`:
  - `buildReportData(period, sections)` — фильтрует данные из `src/lib/mock-data.ts` по периоду (7→последние недели, 30/90→месяцы, ytd→всё).
  - `exportPDF(data, options)` — через `jspdf` + `jspdf-autotable`, с шапкой и опциональной сводкой.
  - `exportCSV(data)` — многосекционный CSV (заголовок секции + таблица + пустая строка).
  - Имя файла: `report-{period}-{YYYY-MM-DD}.{pdf|csv}`.
- Зависимости: `bun add jspdf jspdf-autotable`.
- Обновить `src/components/topbar.tsx`: заменить статичный `<Button>Отчёт</Button>` на триггер диалога; диалог получает текущий `period` как initial value.
- Только фронтенд, БД/серверные функции не трогаем.
